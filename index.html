  • CommentsApp project github

    "Комментарии"(в рамках тестового задания)

    На странице расположена статья. Требуется создать для этой статьи собственную систему комментариев.

    Каждый комментарий состоит из следующих блоков: Аватар автора комментария, Имя автора комментария, текст комментария, дата написания комментария, ссылка "Ответить", кнопки рейтинга "+" и "-", текущий рейтинг комментария.

    Нажатие на "+" или "-" изменяют рейтинг комментария на 1. В случае, когда рейтинг комментария становится ниже "-10", комментарий должен свернуться в одной строчку "Открыть комментарий".

    Дата комментария должна быть записана в виде временного интервала, прошедшего с момента написания комментария (см. приложенный макет). Шаг интервала: от 0 до 1 часа - минуты, от 1 часа до 1 суток - часы, от 1 суток - дни.

    Комментарии должны добавляться без перезагрузки страницы. Текст комментария должен поддерживать разметку Markdown.

    На любой комментарий можно оставить ответ. При этом форма добавления комментария должна быть перемещена к комментарию, для которого пишется ответ. Максимальный уровень вложенности комментариев - 3. Вложенные комментарии можно сворачивать.

    Форма отправки комментария должна иметь минимальную валидацию: на отсутствие значений, на формат почты.

    Требования:

    • Для реализации логики разрешается использовать любые javascript-фреймворки или библиотеки.
    • Для верстки запрещается использовать фреймворки (bootstrap, foundation, pure и другие).
    • Верстка должна быть выполнена с использованием одного из препроцессоров: LESS, SASS/SCSS, Stylus.
    • Для обработки markdown'a использовать сторонние расширения запрещается.
    • Скрипты должны быть собраны в один файл, стили - в один файл стилей, вспомогательные картинки - в спрайт.
    • Выполненное задание должно располагаться на github.com. В репозитории должна быть история коммитов (то есть запрещается выкладывать уже готовый проект).
    • Готовое задание быть выложено на github pages. Плюсом будет наличие комментариев в формате JSDoc.
    • Дополнительно. Код нужно покрыть тестами.
